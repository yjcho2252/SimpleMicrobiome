# Release Notes - 2026-04-25

## Release Metadata
- Branch: `main`
- Scope: Beta/heatmap/taxa-comparison improvements, labeling consistency, prevalence workflow updates, and stability fixes

## Summary
This release improves ordination and comparison workflows with clearer labeling, prevalence-focused analysis paths, and stronger UI behavior consistency. It also fixes taxa selection state loss in taxa comparison and hardens relative-abundance handling for zero-sum edge cases.

## Changes

### 1) Beta diversity enhancements (`modules/mod_beta.R`)
- Added `Jaccard` distance option and fully integrated it into ordination/permanova labels and behavior.
- Implemented binary presence/absence conversion when Jaccard is selected.
- Clarified Aitchison display label as `Aitchison (CLR log(x+1) pseudocount)`.
- EnvFit overlay improvements:
  - Removed `Taxa::` prefix from taxa labels.
  - Removed group-name prefixes from factor arrow labels.
  - Applied bold style to EnvFit labels.
  - Added a visible origin marker at `(0,0)` for arrow vectors.
- Set fixed random seed for EnvFit permutation reproducibility.

### 2) Heatmap prevalence workflow (`modules/mod_heatmap.R`)
- Added association mode selector:
  - `Abundance-based`
  - `Prevalence-based (binary)`
- In prevalence mode:
  - Uses binary-aware association/testing paths.
  - For categorical groups, cell values are delta prevalence (`in-group - out-group`).
  - Disables non-applicable controls (`Data transform`, `Correlation method`, `Value scale`).
- Updated status and legend text to report the actual association/test context used.
- Refined legend title formatting (compact, multi-line, centered, reduced font size).

### 3) Taxa comparison reliability and scale behavior (`modules/mod_taxa_comparison.R`)
- Fixed taxa selection reset when changing secondary group after preselecting multiple taxa.
- Updated taxa filtering p-value logic (`show_p_lt_0_05_only`) to align with selected test mode and paired setting.
- Improved y-scale behavior:
  - Prevents CLR negative-value clipping in boxplots.
  - Aligns bar/box y-axis interpretation while rendering bars from axis baseline.
- Improved axis label readability via pretty breaks and simplified numeric labels.
- Restored boxplot outlier point visibility.
- Increased auto width scaling factor for faceted plots (`*10` to `*20`).

### 4) Bar plot normalization guard (`modules/mod_barplot.R`)
- Added zero-sum-safe normalization branch to avoid invalid relative abundance values for empty/zero-sum samples.

### 5) Alpha legend clarity (`modules/mod_alpha.R`)
- Updated legend significance text to reflect:
  - 2-group comparison wording
  - 3+ group pairwise comparison wording
  - selected statistical method and multiple-testing correction context

## Notes
- No migration steps are required.
- Existing workflows remain compatible, with improved interpretability and robustness.
