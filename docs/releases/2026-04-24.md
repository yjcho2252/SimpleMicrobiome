# Release Notes - 2026-04-24

## Release Metadata
- Commit: `f4550ff`
- Branch: `main`
- Scope: Grouping flow improvements, UI wording updates, layout fix, and placeholder text readability

## Summary
This release introduces a two-stage grouping workflow (Primary and Secondary) across key analysis modules, improves wording clarity for group selection, fixes table/legend overlap issues, and reduces placeholder message font size for better readability inside plot areas.

## Changes

### 1) Primary/Secondary grouping workflow
Implemented a staged grouping flow in the following modules:
- `modules/mod_ancom.R`
- `modules/mod_maaslin2.R`
- `modules/mod_randomforest.R`
- `modules/mod_sparcc.R`
- `modules/mod_spieceasi.R`
- `modules/mod_heatmap.R`

Behavior:
- Primary grouping variable + primary level acts as a pre-filter.
- Secondary grouping variable is used for comparison.
- If Secondary is set to `None`, comparison falls back to Primary-only mode.

### 2) Group selection wording update
Updated labels in:
- `modules/mod_ancom.R`
- `modules/mod_maaslin2.R`

Text changes:
- From: `Secondary group levels to include`
- To: `Comparison groups (levels)`

Placeholder changes:
- From: `Select two or more levels`
- To: `Select two or more groups to compare`

### 3) Table/legend overlap fix
Fixed table output overlapping legend/status areas in:
- `modules/mod_ancom.R`
- `modules/mod_maaslin2.R`

Approach:
- Wrapped table output in a dedicated container with stable stacking and spacing.
- Added explicit positioning/z-index handling for legend/status blocks.

### 4) Placeholder text readability
Reduced placeholder message font size in plot-empty/wait states across:
- `modules/mod_alpha.R`
- `modules/mod_ancom.R`
- `modules/mod_barplot.R`
- `modules/mod_beta.R`
- `modules/mod_biplot.R`
- `modules/mod_heatmap.R`
- `modules/mod_maaslin2.R`
- `modules/mod_randomforest.R`
- `modules/mod_sparcc.R`
- `modules/mod_spieceasi.R`
- `modules/mod_taxa_comparison.R`

Applied style:
- `graphics::text(..., cex = 0.85)`

## Notes
- No migration steps are required.
- Existing analysis workflows continue to work, with improved group-selection flexibility and clearer UI behavior.
