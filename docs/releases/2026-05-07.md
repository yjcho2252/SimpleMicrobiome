# Release 2026-05-07

## Summary
- Updated subgroup primary-level dropdown behavior in Heatmap to use the number of distinct levels (not per-level sample count).
- Added optional ordered trend testing support in ANCOM-BC2 workflows.
- Added run cooldown/guard to SpiecEasi network execution.
- Refined Taxa Comparison trend handling and q-value-based filtering/annotation.
- Simplified top-page update history into versioned release highlights.

## Changed
- Heatmap (`modules/mod_heatmap.R`):
  - Updated `Primary level to include` choices logic for subgroup mode.
  - Primary-level choices are now shown only when the number of distinct levels is fewer than `10`.
  - Updated status text to match the distinct-level-count rule.
- ANCOM (`modules/mod_ancom.R`):
  - Added `Enable ordered trend test` option.
  - Trend testing now runs only when enabled and when at least 3 levels are selected.
  - Added ANCOM-BC2 trend control setup and passed it via `trend_control`.
  - Included trend results in processed output (`trend_*` columns when available).
- SpiecEasi (`modules/mod_SpiecEasi.R`):
  - Added run-state guard and 10-second cooldown for `Run SpiecEasi`.
- Taxa Comparison (`modules/mod_taxa_comparison.R`):
  - Changed trend group type options to explicit `Categorical`/`Continuous`.
  - Updated significance-only filter wording and behavior to q-value based filtering.
  - Added multiple-testing adjustment integration for trend-related outputs.
  - Updated trend facet annotation labels from p-value to q-value.
- Top page (`modules/mod_top.R`):
  - Replaced long chronological update list with concise versioned highlights.
