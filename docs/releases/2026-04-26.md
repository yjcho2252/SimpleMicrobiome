# Release Notes - 2026-04-26

## Release Metadata
- Branch: `main`
- Scope: Primary-level `All` filter alignment, Random Forest result/status UX split, and plotting/status refinements

## Summary
This release aligns primary-level filtering behavior across analysis modules and improves Random Forest usability by separating status and model outputs into dedicated UI areas. It also improves SHAP plot font scaling consistency and includes additional stability/UX updates in heatmap, preprocessing, and taxa comparison flows.

## Changes

### 1) Primary-level `All` behavior alignment
- Updated ANCOM, MaAsLin2, and Random Forest to follow the same `Primary level` handling used in Heatmap:
  - Added `All` to the `2. Primary level to include` options.
  - Set default selection to `All`.
  - Skipped strict primary-level equality filtering when `All` is selected (while still excluding missing primary labels where applicable).
- Files:
  - `modules/mod_ancom.R`
  - `modules/mod_maaslin2.R`
  - `modules/mod_randomforest.R`

### 2) Random Forest result/status separation and status clarity
- Added a dedicated `Result` tab for model status/metrics output.
- Restored the bottom `Random Forest Status` section and limited it to sample selection context only.
- Expanded status content to include training target context (e.g., class groups used for training in classification).
- Removed verbose SampleID preview from status and replaced it with analysis-goal oriented text.
- Files:
  - `modules/mod_randomforest.R`

### 3) Random Forest SHAP font scaling consistency
- Updated SHAP bar plot text sizing to follow `Base Font Size` rather than fixed font values:
  - Taxa labels
  - Class column labels
  - Bar-axis label text
- Applied the same scaling behavior to both on-screen rendering and PNG download output.
- File:
  - `modules/mod_randomforest.R`

### 4) Heatmap and metadata-resolution refinements
- Added robust metadata column resolution fallback by normalized name matching.
- Ensured `Primary level` choices include `All` and stay synchronized when group variable changes.
- Improved prevalence/z-score mode labeling and status detail reporting.
- File:
  - `modules/mod_heatmap.R`

### 5) Additional UX and plotting adjustments
- Preprocessing: ensured full-row default selection is applied when leaving the tab without explicit manual selection.
- Taxa comparison: improved selected taxa ordering stability and refined bar/scale handling for faceted comparisons.
- Files:
  - `modules/mod_preprocessing.R`
  - `modules/mod_taxa_comparison.R`

## Notes
- No migration steps are required.
- Detailed changes are also recorded in `CHANGELOG.md`.
