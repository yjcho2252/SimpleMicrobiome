## Update (Taxa/Beta UX and Clustering Overlay)

### Summary
- Extended taxa comparison trend-line workflows with scatter mode and facet-level trend p-value display.
- Updated beta diversity clustering UX with optional cluster-color overlay on ordination dots.
- Removed the standalone Correlation Plot module from app navigation and server wiring.

### Changed
- Taxa comparison (`modules/mod_taxa_comparison.R`):
  - Added trend-line controls near the Within-Subject Pairing controls:
    - `Show trend line`
    - `Trend Line Method` (`Spearman`, `Pearson`, `Linear regression (lm)`).
  - Added `Scatter plot` to plot type options and auto-switched to scatter when trend-line display is enabled.
  - Added facet-level trend p-value text (`p-value = ...`) when trend-line display is enabled.
  - Enabled smoothing interval display (`se = TRUE`) for trend lines.
  - Increased top y-axis expansion and moved trend p-value text placement upward to reduce overlap.
  - Included `SampleID` in Primary Group choices, while defaulting to a non-`SampleID` column when available.
- Beta diversity (`modules/mod_beta.R`):
  - Included `SampleID` in Primary Variable choices, while defaulting to a non-`SampleID` column when available.
  - Added clustering option `Overlay cluster colors on dots`.
  - When enabled and clustering results are present, overlays cluster-colored dots and shows a `Cluster` legend category.
  - Expanded clustering result text to clarify:
    - average silhouette width is a sample-level overall mean,
    - optimal `k` is selected by maximizing average silhouette width.
- App wiring (`app.R`):
  - Removed `Correlation Plot` (`mod_scatter`) nav panel and server call after module removal.

### Removed
- Removed scatter/correlation module file:
  - `modules/mod_scatter.R`
